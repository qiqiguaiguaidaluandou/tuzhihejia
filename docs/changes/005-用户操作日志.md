# 005 用户操作日志

- 状态：已完成
- 日期：2026-06-02
- 关联提交：（待回填）

## 需求 / 改动

为用户操作加日志：点"回传"等按钮后记录一条，字段含**操作按钮、操作电脑IP、时间、表单名称**。
查看权限分级：

- **管理员**看所有电脑的操作日志（直接在服务器侧查看，不在 App 内）。
- **操作员**在 App 内有一个"操作日志"页，**只看自己**的记录。

本期只记 **①整批回传** 和 **②异常补处理单行回传** 两个动作，后续按需扩展。

## 方案

- 集中上报：日志必须能让管理员在一处看到所有电脑，故客户端每次操作 **POST 到后端集中存**；不做纯本地文件。
- 一条记录字段：`Operation`(操作按钮文案) / `ClientIp`(本机局域网 IPv4) / `OperatedAt`(客户端时间) / `FormName`(批次文件夹名) / `Flow` / `EmployeeId`。
  - **工号由后端从令牌盖章写入**（防伪造），同时作为"操作员只看自己"的过滤键。
- 记录时机：回传/补回传**成功之后**记一条（失败的回传不算一次有效操作）。
- 记录方式：**fire-and-forget**，记日志失败只吞掉写调试输出，**绝不打断/拖慢回传本身**，也不弹提示打扰操作员。
- 后端存储：落**文件**（JSONL，按月一个 `operations-yyyyMM.log`，每行一条 JSON）。管理员直接打开文件即可查看所有电脑的操作；与回传审计 `IAuditStore`（只记成功回传供补拉判据）分开。上线可改落 PostgreSQL。
- 查看：操作员页调 `GET /api/oplog/mine`，后端按令牌工号过滤——只能看到自己。离线 Mock 模式下进程内内存留存（当前会话内可见）。
- 入口：侧边栏"系统"组新增"操作日志"项（与采集计划/设置并列，所有操作员可见）。

## 实现

**共享契约（TZHJ.Core）**

- `Contracts/Http/HttpDtos.cs`：新增 `OperationLogEntry`（wire DTO，前后端共用）、`OperationLogListResponse`。
- `Contracts/IOperationLogGateway.cs`：新增网关契约 `RecordAsync` / `ListMineAsync`。

**后端（TZHJ.Gateway）**

- `Stores/IOperationLogStore.cs` + `Stores/FileOperationLogStore.cs`：文件实现（JSONL 追加、按工号全量扫描过滤、`lock` 串行写）；`OperationLogOptions`（目录可配，默认 `logs`）。
- `Endpoints/ApiEndpoints.cs`：新增 `POST /api/oplog`（工号以令牌盖章后落盘）、`GET /api/oplog/mine`（按令牌工号返回本人记录）。
- `Program.cs`：绑定 `OperationLog` 配置节、注册 `IOperationLogStore`。
- `appsettings.json`：新增 `OperationLog:Directory`。

**客户端网关（TZHJ.Infrastructure）**

- `Gateways/Http/HttpOperationLogGateway.cs`：真链路（POST/GET，经 `AuthTokenHandler` 带令牌）。
- `Gateways/Mock/MockOperationLogGateway.cs`：离线占位（内存留存）。
- `DependencyInjection.cs`：Mock / Http 两处各注册一行。

**客户端（TZHJ.App）**

- `Services/MachineInfo.cs`：取本机局域网 IPv4（优先已启用非回环网卡，回退机器名解析，取不到返回 null）。
- `Services/OperationLog.cs`：记录助手，统一拼装记录 + fire-and-forget 上报 + 吞错。
- `ViewModels/BatchWorkViewModel.cs`：构造注入 `IOperationLogGateway`；`Submit` 成功分支记一条（操作=`回传到{SRM/EBS}`，表单=批次 `FolderName`）。
- `ViewModels/ExceptionResolveViewModel.cs`：同上；`Upload` 成功分支记一条（操作=`重新回传到{...}`，表单=来源批次名）。
- `ViewModels/OperationLogViewModel.cs` + `Views/OperationLogView.xaml(.cs)`：只读列表页（时间/操作按钮/表单名称/操作电脑IP + 刷新），仿异常池页样式。
- `App.xaml`：VM→View 模板映射。
- `Services/INavigationService.cs`：`ToOperationLog()`。
- `ViewModels/ShellViewModel.cs`：侧边栏"系统"组新增"操作日志"（图标 `Symbol.History`）。

不变：回传/补回传业务逻辑、行状态机、提交闸门、回传审计、其余页面。

## 验证

`dotnet build TZHJ.sln -p:EnableWindowsTargeting=true` 0 错 0 警。
WPF 真机待验：真链路回传/补回传成功后服务器 `logs/operations-*.log` 多一行；"操作日志"页只显示本人记录、四列齐全；断网/后端不可达时回传照常成功、不报日志错误。
