## 需求 / 改动

`docs/接口文档.md` 给出了从 EBS 取基础数据的两个接口：

- **图纸核价** `P_IFACE_CODE=CUX_AI_DRW_COST`：返回数据带 `GROUP_NAME`，按组分别存。
- **机加中心挑图** `P_IFACE_CODE=CUX_AI_MACH_DRW`：不分组，整批存。

改前取数全部由占位防腐层 `FakeDataSource` 造假数据。本次把真实 EBS 取数接进来，替换/补齐防腐层接缝 `IEbsPlmSource` 的实现（路线图 B1）。图纸文件与"是否变更"来自 PLM，PLM 接口文档尚未提供，本次不做。

## 方案

- **接缝不动上层**：在防腐层 `IEbsPlmSource` 后面加真实实现 `EbsPlmSource`，用 `Ebs:Enabled` 开关切换（默认 false 仍走 `FakeDataSource`，不影响本地验证）。端点、采集服务上层、客户端、字段 schema 大体不动。
- **鉴权**：对方要求带签名 JWT 放进 `Authorization`（`Bearer` 前缀）。复刻其 `JWTUtil.CreateJWTToken()`：HS256，claims `iss/exp(+15m)/sub("")/aud("")/nbf(-10m)`，10 位秒级时间戳。**手写 base64url + HMACSHA256**，不走 Microsoft.IdentityModel——一为与对方 jwt-dotnet 输出一致，二为避开 Microsoft 库对对称密钥长度的强制校验（对方 secret 可能较短）。`nbf-10m / exp+15m` 的宽窗用于容忍两端时钟偏差。
- **两接口 URL 不同**：分别配 `PricingUrl` / `DrawingUrl`，按流程选；`P_IFACE_CODE` 仍随包体发。
- **分组改为跟随数据**：旧 `DataIngestionService` 写死 `组1/组2` 预设组再取数；真实里组别由响应每行 `GROUP_NAME` 带出。改成"取整窗一次 → 按 `GROUP_NAME` 拆组落盘"（挑图不分组，归 `Center`）。`GroupName` 是不透明键（文件夹名 = DB = 权限范围键），完整组名原样使用，无硬编码比较。
- **采集窗口改为项目排程**（替换原"每5分钟滚动1小时整点窗"）：按固定时刻触发、各带跨天范围——
  - 图纸核价：**10:00** 取「昨天15:31~今天09:30」；**16:00** 取「今天09:31~15:30」。
  - 机加挑图：**10:00** 取「昨天18:01~今天09:30」；**15:00** 取「今天09:31~14:30」；**18:30** 取「今天14:31~18:00」。
  - 触发时刻比窗口结束晚约 30 分钟（数据落定缓冲）；各流程窗口首尾相接整天闭合。服务每 5 分钟轮询，回看「昨天+今天」补采，已采批次跳过、不重复调接口（重启可补采）。
- **字段映射**（详见 `docs/接口文档.md`，用户逐字确认）：
  - 核价：`ITEM_CODE→materialCode`(主键)、`ITEM_MODEL→model`、`ITEM_NAME→name`、`ITEM_DESC→materialDesc`(**新增列**)、`TTL_QTY→demandQty`、`GROUP_NAME→分组`。
  - 挑图：`SEQ_ID→ebsId`(主键)、`ORGANIZATION_CODE→invOrg`、`SOURCE_DOC_NUMBER→sourceNo`、`PROJECT_CODE→project`、`PROD_CODE→productLine`、`INTER_PROJECT_CODE→planNo`、`ITEM_CODE→materialCode`、`ITEM_DESC→materialDesc`、`OFFSET_CUR_QTY→currentQty`、`CREATION_DATE→createDate`、`NEED_BY_DATE→demandDate`、`EMPLOYEE_NAME→applicant`、`REMARKS→remark`、`DEPARTMENT→deptDesc`。`ITEM_MODEL` 丢弃；`DEPARTMENT` 接口暂未返回 → 容错读取，缺则空。

## 实现

**新增**（`src/TZHJ.Gateway/AntiCorruption/`）：
- `EbsOptions.cs`：`Ebs` 配置段（`Enabled`、`PricingUrl`/`DrawingUrl`、`JwtSecret`/`JwtIssuer`、`AuthScheme`、两个 `IfaceCode`）。
- `EbsTokenProvider.cs`：手写 HS256 JWT，复刻对方 token。
- `EbsPlmSource.cs`：`IEbsPlmSource` 真实实现。封包（`P_BATCH_NUMBER="AI"+时间戳`、`P_REQUEST_DATA` 为 JSON 字符串、时间窗由调用方按项目规则给定）→ POST → 判 `X_RETURN_CODE=="S"` → 二次解析 `X_RESPONSE_DATA` 取 `DATA` → 按映射转 `SourceRow`。`OpenDrawingAsync` 暂返回 null（PLM TODO）。

**改动**：
- `Core/Schemas/FieldSchemas.cs`：核价加 `materialDesc`（"物料描述"）列，Order 重排。
- `Gateway/AntiCorruption/SourceModels.cs`：`SourceRow` 加 `GroupName`。
- `Gateway/AntiCorruption/FakeDataSource.cs`：核价造数按行交替打 `GroupName`（组1/组2）+ 填 `materialDesc`，与新分组流程对齐。
- `Gateway/DataIngestionService.cs`：删除写死的 `组1/组2` 与旧的滚动整点窗（`SeedHistoryAsync`/`SeedLatestAsync`）；新增 `Schedule` 排程表 + `CheckSchedulesAsync`（按固定时刻触发、回看两天补采）、`IngestWindowAsync`（批次级预检避免重复取数 → 取整窗 → 按 `GROUP_NAME` 分组）+ `PersistGroupAsync`（落盘/注册）。
- `Gateway/Program.cs`：绑定 `EbsOptions`、注册 `EbsTokenProvider`；`Ebs:Enabled` 为真时 `AddHttpClient<IEbsPlmSource, EbsPlmSource>()`，否则仍用 `FakeDataSource`。回传 `ISubmitSink` 仍 Fake。
- `Gateway/appsettings.json`：加 `Ebs` 段（占位，`Enabled:false`）。
